# Protocol Status

Protocol: NPPP — Notarization Proof Packet Protocol  
Version: v1  
Status: Active

---

## Current Capabilities

NPPP v1 provides replay-verifiable evidence packets.

The protocol allows systems to:

• canonicalize digital artifacts  
• produce deterministic evidence bundles  
• bind bundles to SHA-256 proof strings  
• replay verification independently  

Verification can be performed without trusting the original operator.

---

## Production Availability

The protocol opens publicly on:

**March 26, 2026**

Initial network capabilities include:

• notarization of canonical digital artifacts  
• replay verification via CLI or API  
• proof ledger entries for each notarization  

---

## Limitations

NPPP v1 verifies integrity of evidence bundles.

The protocol does not assert:

• truthfulness of submitted artifacts  
• legal ownership of underlying assets  
• external system correctness  

---

## Version Stability

The NPPP v1 proof format is frozen.

Future improvements will occur in later protocol versions.

Existing proofs will remain verifiable indefinitely.
